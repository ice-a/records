# Paste Logbook (Next.js + MongoDB Atlas + Vercel)

Paste Logbook is a note app with:
- markdown rendering
- per-record timestamps
- tag classification
- search + tag filters
- one-click AI summary
- password gate before access

## Features

- New record area at the top.
- Save creates one Mongo document per record.
- Saved records are read-only by default.
- Click `Edit` to show the edit form.
- Each record stores:
  - `createdAt`
  - `updatedAt`
  - `content`
  - `tags`
  - `summary` (optional, generated by AI)

## Run locally

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Environment variables

Copy `.env.example` to `.env.local`:

```bash
MONGODB_URI=mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
MONGODB_DB=paste_logbook
MONGODB_COLLECTION=records
APP_PASSWORD=change_this_password
AUTH_SALT=change_this_salt
ALLOWED_DEV_ORIGINS=http://localhost:3000,http://192.168.28.11:3000

LONGCAT_API_KEY=YOUR_APP_KEY
LONGCAT_MODEL=LongCat-Flash-Chat
LONGCAT_API_URL=https://api.longcat.chat/openai/v1/chat/completions
```

`LONGCAT_API_KEY` and `LONGCAT_MODEL` match your API demo settings.
`APP_PASSWORD` is required for login gate.
`AUTH_SALT` is used to derive the secure auth cookie token.
`ALLOWED_DEV_ORIGINS` is optional but recommended for LAN access in `next dev`.

## API

- `GET /api/notes`
  - returns saved records sorted by `createdAt` desc
- `POST /api/notes`
  - body: `{ content: string, tags: string[] }`
  - creates one record
- `PATCH /api/notes/:id`
  - body can include `{ content, tags, summary }`
  - updates one record
- `POST /api/notes/ai-summary`
  - body: `{ content: string }`
  - returns AI-generated summary

## Deploy to Vercel

1. Push this project to GitHub/GitLab/Bitbucket.
2. Import the repo in Vercel.
3. Add all env vars from above in `Settings -> Environment Variables`.
4. Redeploy.
